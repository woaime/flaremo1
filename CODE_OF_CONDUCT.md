# Code of Conduct

FlareMo uses the Contributor Covenant Code of Conduct.

## Our Standard

Project spaces should stay focused, respectful, and useful. Good participation includes:

- Discussing ideas and tradeoffs with concrete reasoning.
- Reporting problems with enough detail for others to reproduce them.
- Disagreeing with code, design, or direction without attacking people.
- Respecting maintainers' decisions about project scope.

Unacceptable behavior includes harassment, personal attacks, threats, sexualized language, doxxing, spam, and repeated disruption after maintainers ask for it to stop.

## Scope

This applies to the FlareMo repository, issues, pull requests, discussions, release channels, and any project-run community space.

## Enforcement

Maintainers may edit or delete comments, close issues, block users, or restrict participation when behavior harms the project.

Report conduct problems through the maintainer contact listed in the GitHub profile, or by opening a private security report if the situation also includes sensitive information.
